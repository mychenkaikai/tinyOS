- [x] 规格明确第一阶段范围只覆盖最小启动链路、基础内核能力、简单显示和 GUI MVP
- [x] 规格明确排除完整用户态、POSIX、SMP、GPU 驱动、完整网络栈等高复杂度首阶段目标
- [x] 规格定义了 `boot`、`arch`、`platform`、`kernel` 等核心分层，避免后续实现混杂

- [x] 规格给出了 MCU 子集的收敛方式，而不是空泛地声称“以后支持 MCU”
